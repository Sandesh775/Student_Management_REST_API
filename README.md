# 🚀 Mini Project 1: Student Management REST API

## 📌 Scenario
A local training institute has requested a backend system to manage student records. Currently, they require a robust REST API that allows staff to register students, retrieve records, update information, and delete students. 

Since the institute is actively testing the system, this backend utilizes **MySQL** without authentication or a frontend interface. The primary goal of this project is to demonstrate clean architecture, proper validation, layered design, and strict adherence to REST principles.

---

## 🎯 Objective
Build a complete, production-ready **Spring Boot CRUD API** leveraging clean code practices, explicit validation rules, and comprehensive global exception handling.

---

## 🛠️ Tech Stack & Dependencies
* **Framework:** Spring Boot (Spring Web, Spring Data JPA)
* **ORM:** Hibernate
* **Database:** MySQL
* **Validation:** Hibernate Validator / Bean Validation API
* **Error Handling:** ControllerAdvice / Global Exception Handling & `ResponseEntity`

---

## 🗄️ Database Schema & Student Entity
The system manages a `Student` entity with the following attributes and database-generated IDs:

| Field | Type | Validation Rules | Description |
| :--- | :--- | :--- | :--- |
| `id` | `Long` | Primary Key (Auto-Generated) | Unique identifier generated by the database. |
| `fullName` | `String` | `@NotBlank` | The full name of the student; cannot be empty. |
| `email` | `String` | `@NotBlank`, `@Email` | Student email; must be a valid, unique email address. |
| `age` | `Integer` | `@NotNull`, `@Min(0)` | Must be a positive integer. |
| `course` | `String` | `@NotBlank` | The course the student is enrolled in. |

> 💡 *Note: You are encouraged to strengthen these validation rules using additional built-in annotations if necessary.*

## 🏗️ Architecture & Layered Design
The project strictly follows a layered architecture to maintain a separation of concerns. **No business logic should exist inside the Controller layer.**

```text
Student Management API
├── 📁 controller      # Handles incoming HTTP requests and REST endpoints
├── 📁 service         # Houses core business rules and transactional logic
├── 📁 repository      # Manages direct database communication (Spring Data JPA)
├── 📁 entity          # Defines the database models and mappings
├── 📁 exception       # Contains custom exceptions and the global handler
└── 📄 application.properties
